Module: api/user

User management API endpoints. Handles user CRUD operations, profile picture uploads, and role management. Most endpoints require authentication, with role-based access control.

(async, static) GET(req) → {Promise.<NextResponse>}

GET /api/user Fetches user data. Supports three modes: 1. Check if a specific email exists (users can check their own, admins can check any) 2. Get all users (admin only, via getAllUsers=true query param) 3. Legacy: Get all users (admin only, default behavior)

(async, static) POST(req) → {Promise.<NextResponse>}

POST /api/user Creates a new user or updates an existing user's profile/role. Supports two modes based on the 'action' parameter: 1. action='update': Update user profile (users can update their own, admins can update any) 2. No action or action='create': Create new user (self-registration or admin creation)
